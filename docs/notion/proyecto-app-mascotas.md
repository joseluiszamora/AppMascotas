# Proyecto: App para mascotas

## Descripción general

La app para mascotas será una plataforma móvil comunitaria enfocada en ayudar a encontrar mascotas perdidas y reportar mascotas encontradas.

El objetivo principal es permitir que los usuarios registren sus mascotas, publiquen reportes geolocalizados, reciban alertas cercanas y colaboren entre sí para recuperar mascotas extraviadas.

## Objetivo principal

Crear una plataforma móvil comunitaria para mascotas perdidas y encontradas, donde los usuarios puedan:

- Registrarse con Google.
- Crear su perfil.
- Registrar una o varias mascotas.
- Subir fotos de sus mascotas.
- Reportar mascotas perdidas.
- Reportar mascotas encontradas.
- Visualizar reportes en un mapa.
- Recibir alertas cercanas.
- Compartir reportes por WhatsApp, Facebook u otros canales.
- Colaborar con otros usuarios para recuperar mascotas.

## Enfoque del proyecto

El proyecto debe construirse por etapas.

La primera versión debe enfocarse en validar el problema principal:

> ¿Los usuarios usarán la app para registrar mascotas, reportar pérdidas o avistamientos, y ayudar en casos cercanos?

## Etapas del proyecto

### Etapa 1 — MVP base

Objetivo: construir una versión inicial simple, útil y rápida de lanzar.

Incluye:

- Autenticación con Google.
- Perfil de usuario.
- Registro de mascotas.
- Reportes de mascotas perdidas.
- Reportes de mascotas encontradas.
- Geolocalización.
- Mapa de reportes.
- Filtros básicos.
- Notificaciones push.
- Alertas cercanas.
- Compartir reportes.

### Etapa 2 — Valor avanzado y operación

Incluye:

- IA de reconocimiento visual.
- Matching entre mascotas perdidas y encontradas.
- Contacto entre usuarios.
- Panel administrador.
- Moderación.
- Historial médico de mascotas.

### Etapa 3 — Monetización

Incluye:

- Publicidad segmentada.
- Directorio de servicios.
- Veterinarias.
- Tiendas de mascotas.
- Peluquerías caninas.
- Servicios patrocinados.
- Analíticas y estadísticas.

### Etapa 4 — Comunidad

Incluye:

- Red social de mascotas.
- Publicaciones.
- Fotos y videos.
- Likes.
- Comentarios.
- Seguidores.
- Comunidades por zona.
- Gamificación.

### Etapa 5 — Expansión comercial e IA avanzada

Incluye:

- Marketplace.
- Venta de productos.
- Reservas de servicios.
- Pagos online.
- Delivery.
- Chatbot veterinario.
- Recomendaciones inteligentes.
- Reconocimiento facial animal avanzado.

## Estrategia de monetización

La monetización no debe implementarse en la Etapa 1.

Durante el MVP, la prioridad será:

- Crecer.
- Validar el problema.
- Construir confianza.
- Generar comunidad.
- Obtener reportes reales.
- Medir participación.

El modelo recomendado a futuro será freemium:

- App gratuita para usuarios finales.
- Monetización B2B con veterinarias, tiendas, peluquerías y servicios para mascotas.
- Publicidad segmentada desde la Etapa 3.
- Marketplace y comisiones desde la Etapa 5.

## Stack técnico propuesto

- Flutter para la app móvil.
- Supabase para autenticación, base de datos, storage y realtime.
- PostgreSQL como base de datos.
- Supabase Storage para imágenes.
- Push notifications para alertas.
- IA en etapas posteriores.
- GitHub para control de versiones.
- Notion como documentación funcional.
- Codex como asistente de desarrollo.

## Criterio de éxito de la Etapa 1

La Etapa 1 será exitosa si permite:

- Registrar usuarios y mascotas de forma simple.
- Crear reportes de mascotas perdidas o encontradas en menos de 2 minutos.
- Ver reportes cercanos en un mapa.
- Compartir reportes por redes sociales o mensajería.
- Enviar alertas cercanas.
- Filtrar reportes por ubicación y características.
- Validar interés real de la comunidad antes de invertir en IA y monetización.
